## 핵심 검토 규칙 (15개)

## 개요

이 스킬은 NDA(Non-Disclosure Agreement) 계약서를 검토하고 위험 요소를 분석하는 것을 지원합니다.
5개 이상의 실제 NDA 문서에 대한 법무팀 자문(법무팀 리비전 커멘트)을 분석하여 도출된 **15개 핵심 검토 규칙**을 적용합니다.

> 중요: 문서를 처리할 때 업로드된 PDF/Docx 파일을 read_file 등으로 직접 읽지 마세요. 반드시 memento 도구를 통해 문서 스냅샷(텍스트/메타데이터)을 확보한 뒤, 텍스트 기반으로 검토를 진행하세요. 이는 바이너리 파일을 모델 입력으로 잘못 전달하는 오류를 방지하고, 버전 일관성 및 재현성을 확보하기 위함입니다.

## 0. 문서 취득·전처리 — memento 사용 필수

문서 접근은 항상 다음 절차를 따르세요.

1) 문서 식별
- 사용자 요청/컨텍스트에서 문서명, 경로, 업로드 타임스탬프, 파일 확장자(PDF/DOCX 등)를 추출합니다.
- 경로가 /workspace/uploads/ 하위라고 하더라도, 해당 경로의 파일을 직접 read_file로 열지 않습니다(바이너리 → 모델 입력 오류 방지).

2) memento 호출로 문서 스냅샷 확보
- memento(문서 메모리/스냅샷) 도구를 호출해 대상 문서를 검색/조회합니다.
  - 검색 키: 제목/파일명 힌트, 사용자 요청의 키워드(“NDA”, 당사자명, 날짜), 업로드 경로 힌트.
  - 기대 결과: {doc_id, version, mime, text(추출된 본문 또는 페이지별 텍스트), metadata(페이지 수, 언어 등)}
- memento가 텍스트 스냅샷을 제공하면 그 텍스트를 기준으로 검토를 진행합니다.

3) 후보 문서가 여러 개인 경우(중복 파일명 등)
- 상위 3개 후보를 나열해 최소 확인 질문 1회를 통해 식별합니다.
  - 예: “다음 중 어떤 문서를 검토할까요? ① NDA____.docx(업로드 06:50) ② nda.pdf(업로드 06:51) …”

4) 텍스트가 비어있거나 품질이 낮은 경우
- 우선 memento의 대체 추출(재추출/다른 파서) 옵션을 재시도합니다.
- 여전히 실패 시, extract_document_structure 등 전용 추출 도구를 사용하되 memento가 반환한 안정 경로/핸들을 사용합니다.
- OCR/재업로드가 필요하면 최소 질의로 사용자의 협조를 요청합니다(예: “원문이 스캔본으로 추정됩니다. 텍스트 추출이 어려워 고해상도 PDF 재업로드를 요청드립니다.”).

5) 금지/주의 사항
- DO NOT: read_file로 .pdf/.docx 바이너리를 직접 열어 모델 입력으로 전달(“Invalid value: 'file' … Supported values are: 'text', …” 오류 유발).
- DO NOT: 업로드 경로(/workspace/uploads/…)의 존재만 믿고 같은 경로로 반복 파싱 재시도(no such file 오류 반복). memento를 통해 유효한 핸들/버전을 먼저 확인.
- OK: ls는 후보 존재 확인 용도로만 제한적으로 사용(내용 읽기 금지). 실제 본문 확보는 memento를 통해 수행.

6) 품질 검수 체크리스트(시작 전)
- [ ] 문서 식별자(doc_id/version)가 확정되었는가?
- [ ] 페이지 수/언어/인코딩 등 메타데이터를 확인했는가?
- [ ] 본문 텍스트가 충분히 추출되었는가(표/부록 포함 여부 점검)?

7) 에러 대응 및 재시도 정책
- 파일 미존재/경로 오류: 동일 경로로 재시도하지 말고 memento 재조회 → 후보 식별 → 안정 핸들로 파싱.
- 입력 형식 오류(파일 블랍 전달): 바이너리 전달 금지. 반드시 텍스트(또는 추출된 구조화 텍스트)를 모델에 전달.
- 위 단계 후에도 실패 시, 사용자에게 단 한 번의 명확한 선택지 제시(재업로드/대체 포맷/후보 중 선택) 후 진행.

## 1. 계약 기본 정보 파악

먼저 다음 정보를 확인합니다:

- **계약 당사자**: 회사 간(Co-to-Co)인지, 회사 대 개인(Co-to-Individual)인지
- **계약 유형**: 상호(Mutual) NDA인지, 단일(Unilateral) NDA인지
- **계약 목적**: M&A, 기술협력, 채용/면접, 영업비밀 보호 등

## 2. 15개 핵심 검토 규칙 적용

다음 규칙들을 순서대로 검토합니다. 각 규칙에 대해 **위험도(HIGH/MEDIUM/LOW)**를 평가합니다.

#### Rule 1: 비밀정보 정의의 균형 (HIGH)

비밀정보의 정의가 너무 광범위하거나 지나치게 협소하지 않은지 확인합니다.

- 광범위 정의 ("구두·서면·전자적 형식 등 모든 형태"): 모든 정보가 비밀정보로 취급됨 → **HIGH**
- 협소 정의 ("서면 Confidential 표시된 정보에 한한다"): 구두 공개를 반영하지 못함 → **HIGH**
- 적정 정의: "Confidential 표시가 있거나, 정보의 성질과 공개 상황에 비추어 합리적으로 비밀로 이해되어야 하는 정보"
- 구두 공개 시 30일 이내 서면 확인 조항이 있어야 함
- 구체적 정보 유형 예시 포함 (기술사, 사업계획서, 고객목록 등)

#### Rule 2: 4대 예외 조항 (HIGH)

모든 예외 조항이 포함되어야 합니다. 하나라도 누락되면 **HIGH**:

1. 공개 당시 이미 공지되었거나 수령자의 귀책 없이 사후에 공지된 정보
2. 수령자가 공개자로부터 받기 이전부터 적법하게 보유하고 있던 정보
3. 제3자로부터 비밀유지 의무 없이 적법하게 입수한 정보
4. **독자개발(Independently developed)** — 반드시 포함되어야 함

> 독자개발 예외가 없으면 수령자가 독자적으로 개발한 결과물이 이 NDA로 제약될 수 있습니다.

#### Rule 3: 비밀유지 기간 (HIGH)

- 표준: 계약 종료 후 **2~5년**(M&A는 3년 권장)
- 무기한(Indefinite)은 **HIGH** 위험 — 개인 대상일 때 특히 문제
- 영업비밀(Trade secrets)은 시간제한 면제되어야 함
- 존속기간은 계약 종료일 기준인지, 공개일 기준인지 확인

#### Rule 4: 관련자 제공 조건 (MEDIUM)

- 직전 서면 동의(prior written consent) 방식은 비현실적 → **MEDIUM**
- 적정 방식: 동등 이상의 비밀유지 의무를 부과하는 조건부 제공 허용
- 수령자는 관련자의 위반에 대해 자신의 위반처럼 책임져야 함

#### Rule 5: 책임상한 (HIGH)

- **책임상한 없이 모든 손해에 대한 배상책임은 HIGH** — 반드시 책임상한 조항 필요
- 책임 범위는 직접 손해만 포함 (간접·결과적 손해 제외)
- 고정 금액 상한 또는 거래액/급여의 12개월액 등 명확한 상한 설정
- 상호 NDA의 경우 상호적임

#### Rule 6: 인력 비권유/경업금지 (HIGH)

- 인력 비권유는 **12개월 이내**여야 함 (24개월은 문제)
- 본 계약을 통해 알게 된 임직원만 제한 가능
- 경업금지(Non-compete)는 NDA에 포함 금지 — 별도의 고용계약 부속서에서 처리해야 함

#### Rule 7: 파기확인서 의무 (MEDIUM)

- 비밀정보 반환·파기 의무에 **파기확인서(Destruction Certificate)** 필요
- 수령자 직급 이상 서명, 계약 종료 또는 요청일로부터 30일 이내 전달
- 전자 사본 포함

#### Rule 8: 준거법 및 관할 (MEDIUM)

- 상대방 본국 법률(델라웨어, 캘리포니아 등)은 **MEDIUM** 위험
- 우선순위:
  1. 한국법 + 서울중앙지법 (우리 측 관할) — 최우선
  2. SIAC (Singapore International Arbitration Centre) — 중립 대체안
  3. 기타 중립 관할 (HKIAC 등)

#### Rule 9: 통지 조항 (LOW)

- 양쪽 당사자의 연락처, 이메일, 담당자 명시
- 이메일 통지 시 **간주 수신 규정** (5영업일 경과 시 수신으로 간주)
- 계약 발효일 명시

#### Rule 10: 비차단선언/거래의무없음 (HIGH)

- 본 계약이 거래 체결 의무를 발생시키지 않음 명시
- 당사자 어느 측도 언제든지 협상 종료 가능
- 우선교섭권이나 독점적 교섭권 조항 부재 확인

#### Rule 11: 잔존 지식 조항 (MEDIUM)

- 직원의 비조력 기억(retained in unaided memory)에 보존된 일반 지식·기술 사용 허용
- 구체적 비밀정보나 문서는 제외됨 명시

#### Rule 12: 회사 대 개인 vs 회사 간 (HIGH)

- 상대방이 개인(자연인, 후보자)인 경우:
  - 회사용 템플릿의 부적절 조항 제거 (경업금지, 과도한 위약금 등)
  - 인력 비권유/경업지지는 별도의 고용계약에서 처리
  - 위약금은 개인 소득 대비 적정 수준 (연봉의 30~50%)
  - 무기한 비밀유지 의무 문제 — 기간 설정 필수

#### Rule 13: 상호 vs 단일 NDA (MEDIUM)

- 상호 NDA로 명기되었으나 정보 흐름이 비대칭인 경우:
  - 실제 정보 흐름을 평가하여 단일 NDA로 변경 고려
  - 상대방이 우리 공개 정보를 근거로 부당한 주장을 할 수 있는지 판단

#### Rule 14: 통제권 변경 조항 (MEDIUM)

- 안티-할당 조항에 M&A/통제권 변경 시 어떻게 되는지 명시
- 선택: (a) 통제권 변경 시 자동 종료, 또는 (b) 변경 전 공개자 동의 필요

#### Rule 15: 개인정보(PII) 처리 (HIGH)

- 채용/면접 맥락에서 개인정보가 공유되는 경우:
  - 개인정보보호법(PIPA) 준수 조항 포함
  - 또는 면접 과정에서 개인정보는 익명화/가명화하여 공유

### 2.5. 위험도별 그룹핑 및 정렬 (필수)

- 각 항목의 위험도를 평가한 후, **HIGH → MEDIUM → LOW 순으로 그룹을 나누어** 결과를 작성합니다.
- 각 그룹 내에서는 아래 기준으로 정렬합니다:
  1) 법적/금전적 노출이 큰 항목(예: 책임상한 부재, 무기한 비밀유지) 우선
  2) 구조적 리스크(예: 4대 예외 미비, 비차단선언 부재)
  3) 운영상 비효율/절차적 리스크(예: 통지 조항 미흡)
- 각 이슈는 반드시 다음 4요소를 포함합니다: [항목/조항명] | [적용 Rule #] | [근거 요약(원문 인용 또는 사실관계)] | [구체적 수정 권고안]

## 3. 결과 작성 — 출력 형식 (위험도별 그룹핑 필수)

다음 템플릿을 **항상 그대로** 사용해 결과를 작성하세요.

```markdown
## NDA 검토 결과

### 계약 기본 정보
- 당사자: A회사 ↔ B회사 (회사 간/회사 대 개인)
- 유형: 상호 NDA/단일 NDA
- 목적: 기술협력/M&A/채용 등

### 위험도별 그룹 요약

#### HIGH 위험 (즉시 수정 필요)
- [비밀유지 기간 | Rule 3] 무기한으로 규정됨 → 권고: 계약 종료 후 3년(영업비밀은 예외)으로 수정
- [책임상한 | Rule 5] 상한 부재, 간접/결과적 손해 포함 → 권고: 직접손해 한정 + 상한액 설정(예: OO원 또는 12개월액)
- [4대 예외 | Rule 2] 독자개발 예외 누락 → 권고: 독자개발 항목 명시 추가

#### MEDIUM 위험 (협상/보완 권장)
- [관련자 제공 | Rule 4] 사전 서면 동의 요구 → 권고: 동등 이상의 비밀유지 의무 부과 조건으로 대체
- [준거법/관할 | Rule 8] 상대방 본국 법 적용 → 권고: 한국법+서울중앙지법 또는 SIAC로 변경
- [파기확인서 | Rule 7] 파기확인서 요구 부재 → 권고: 30일 내 서면 확인서 조항 추가

#### LOW 위험 (보완 권장)
- [통지 조항 | Rule 9] 이메일 간주수신 규정 부재 → 권고: 5영업일 간주수신 명시

### 상세 체크 — 15개 규칙별 테이블 (선택적 부록)
| 항목 | 위험도 | 문제점 |
|------|--------|--------|
| 비밀정보 정의 | MEDIUM | 구두 공개 30일 확인 조항 부재 |
| 4대 예외 | HIGH | 독자개발 예외 누락 |
| 비밀유지 기간 | HIGH | 무기한 — 3년으로 단축 권장 |
| 책임상한 | HIGH | 상한 부재 — 직접 손해 한도 및 금액 상한 필요 |
| ... | ... | ... |

### 주요 수정 제안
1. 제3조 예외 조항에 독자개발 예외 추가
2. 비밀유지 기간을 계약 종료 후 3년으로 변경 (영업비밀 예외 명시)
3. 책임상한 조항 추가 (직접 손해 한도, 상한 OO원)
```
